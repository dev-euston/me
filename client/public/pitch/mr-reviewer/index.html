MR Reviewer

AI-powered code review triage — unified, persistent, and entirely under your control.

AI-Powered GitLab Runs Locally No Cloud Open Source
The Problem

Code review lives in
too many places

⚡
Context is scattered
GitLab tabs, Slack threads, email notifications — every MR is spread across a different surface.
🔁
Feedback gets lost
Review comments pile up with no persistent state. Yesterday's work is invisible today.
🤖
AI has no memory
Every AI session starts from scratch. You re-explain context. You re-triage findings.
📋
No triage workflow
Did you post that comment? Is this suggestion still relevant? No way to know at a glance.
The Solution

One dashboard.
One click.
Zero noise.

All your open MRs across every repository — in a single sidebar. AI analysis on demand. State that persists across sessions.

Multi-repo aggregation — all projects in one sidebar
Open & Draft toggle — switch modes in one click
AI review on demand — one button per MR card
Persistent triage — decisions survive session restarts
AI Review

Click once.
Review intelligently.

Hit REVIEW on any MR card. The AI analyses the full diff in the background — you keep working. When it's done, findings appear inline with file and line context.

Background processing — non-blocking, auto-polled
Inline diff hunks — see exactly which lines triggered each finding
Post to GitLab — turn findings into inline MR comments
Customisable persona — inject your team's conventions & standards
AI Comment Drafting

From finding
to feedback —
one click.

Every review finding comes with a POST button. Hit it and the system drafts a contextual, peer-to-peer comment — using the full diff for context. Review it, tweak it, send it.

1
Click POST on any active finding
Triggers an AI call with the full diff and finding as context.
Loading draft…
2
Draft appears — fully editable
Peer-to-peer tone, no mentoring language. Edit freely before sending.
3
SEND → inline comment on GitLab
Posted as an inline thread at the exact file and line. Item flips to POSTED.
Triage Workflow

Four states. Complete clarity.

ACTIVE
Needs attention. Hasn't been actioned yet.
POSTED
Comment sent to the author. Waiting on a response.
IGNORED
Not worth pursuing. Dismissed without action.
OBSOLETE
Code changed. Finding no longer applies.

State is persisted to disk — cycle through statuses with a single click, come back tomorrow exactly where you left off.
